  • MicroPython is used as a lean and efficient implementation of the Python 3 programming language, optimized to run on microcontrollers and in constrained environments.
  • The wasmtime Python library is used to run WebAssembly binaries, and has binary wheels available.
  • Pyodide is mentioned as an alternative package for running Python using WebAssembly, but it is not supported for server-side Python.
  • The micropython-wasm library uses a custom WASM version of MicroPython, compiled using a script that fetches the MicroPython code and builds it.
  • The library supports persistent interpreter state, allowing variables and functions to stay resident in memory across multiple code execution calls.
  • The library also supports host functions, allowing the Python library to selectively expose functions that can be called by code running in MicroPython.
  • Codex is used to solve the problem of supporting host functions, resulting in 78 lines of C code that are compiled into a 362KB WebAssembly blob.
  • The library has a default "fuel" setting of 20 million, which limits the number of operations a WebAssembly call can execute.
  • The library is available on PyPI, and can be installed and used from Python code.
  • Datasette Agent is mentioned as a use case for micropython-wasm, allowing users to run Python code in a sandbox and interact with the Datasette database.
  • OpenAI's GPT-5.5 model is used to test the micropython-wasm library and attempt to break out of the sandbox.
  • Companies with professional security teams and high-stakes problems are encouraged to use Python in WebAssembly as a sandboxing approach and open source their own solutions.
  • The micropython-wasm library is released as an alpha package, and is not recommended for use in production environments without significant testing and validation.
