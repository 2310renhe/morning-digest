- Architecture: designing the microarchitecture

- RTL Design: writing the hardware description code

- RTL Verification: testing or proving the RTL behaves correctly

- RTL Freeze: locking the design with no more functional changes allowed

- FW/SW Development: firmware and software teams begin bring-up on emulators and FPGA prototypes

- Signoff: running final checks that the design meets requirements

- Foundry Handoff: exporting the finished layout as a GDSII file

- Fabrication: manufacturing wafers in the fab

- Post-Silicon Validation: testing and debugging chips

- System Integration: integrating validated chips into boards and packages

- Production: volume manufacturing and yield optimization.

microgpt (http://karpathy.github.io/2026/02/12/microgpt/)

Extract ALL key points from the source. Cover every major argument, data point, and conclusion. Use bullet points for clarity:

* The microgpt project is a 200-line Python script that trains and infers a GPT model with no dependencies.

* The script is a culmination of multiple projects (micrograd, makemore, nanogpt, etc.) and a decade-long obsession to simplify LLMs to their bare essentials.

* The script contains the full algorithmic content of what is needed: dataset of documents, tokenizer, autograd engine, a GPT-2-like neural network architecture, the Adam optimizer, training loop, and inference loop.

* The dataset used is a list of 32,000 names, one per line, which is used to train the model to generate new, plausible-sounding names.

* The tokenizer is a simple character-level tokenizer that assigns a unique integer to each character in the dataset.

* The autograd engine is implemented from scratch using a single class called `Value`, which wraps a scalar number and tracks how it was computed.

* The `Value` class has a `backward` method that walks the computation graph in reverse topological order, applying the chain rule at each step.

* The model architecture is a stateless function that takes a token, a position, the parameters, and the cached keys/values from previous positions, and returns logits over what token the model thinks should come next in the sequence.

* The model uses a combination of attention and MLP blocks to process the input sequence.

* The attention block projects the current token into three vectors: a query, a key, and a value, and uses these vectors to compute attention weights and a weighted sum of the cached values.

* The MLP block is a two-layer feed-forward network that projects up to 4x the embedding dimension, applies ReLU, and projects back down.

* The model uses residual connections to add the output of each block back to its input.

* The final hidden state is projected to vocabulary size by `lm_head`, producing one logit per token in the vocabulary.

* The script is available on GitHub and can be run as a Google Colab notebook.

* The author has also created a video explaining the autograd engine and the `Value` class.

* The script has been simplified to its bare essentials, with no unnecessary dependencies or code.

* The model is trained using a simple Adam optimizer and a cross-entropy loss function.

* The script generates new, plausible-sounding names by sampling from the trained model.

* The generated names are printed to the console, along with their corresponding token IDs and positions in the sequence.

A Validated Volatility-Volume-Gap Classifier for Regime Identification in MNQ Intraday Data(https://arxiv.org/abs/2605.11423)

  • The paper constructs a composite day-classification system for Micro E-Mini Nasdaq 100 futures (MNQ) using three pre-market observable conditions: first-30-minute return magnitude, overnight gap magnitude, and abnormal opening-bar volume
  • The system is validated using 947 regular trading days of five-minute data from 2021-2025
  • Classifier-positive days exhibit statistically distinct intraday behavior, including directional morning drift followed by systematic late-session reversal
  • Despite these characteristics, all tested directional trading strategies fail institutional validation standards after transaction costs and multi-year consistency requirements are applied
  • The highest-performing configuration achieves T = 1.46 and mean net +7.80 points but fails year-stability criteria

Fill-Side Non-Retail Trading on Polymarket: An Empirical Study of Behavioral Tiers and Microstructure Signatures Under Quote-Attribution Constraints(https://arxiv.org/abs/2605.11640)

  • The paper studies non-retail participation on Polymarket using an empirical run on the PMXT v2 archive over 2026-04-21 through 2026-04-27
  • The study includes 13,356,931 OrderFilled events, 77,204 addresses with five+ fills, and 43,116 markets
  • Polymarket's off-chain CLOB architecture renders address-level quote-lifecycle attribution permanently unavailable
  • Density-based clustering on the fill-side vector produces a single dense cluster with zero noise: fill-side behavior in the empirical window is uni-modal under the six-feature vector
  • Robust retail vs non-retail separation is achievable through clustering-independent feature-tier stratification: whale-tier, high-frequency-operator, and power-trader tiers jointly hold 81.4% of total notional across 12.6% of addresses

RED-2400: A Public Benchmark of Algorithmically-Rejected Trading Events with Outcome Labels(https://arxiv.org/abs/2605.12151)

  • RED-2400 is a public benchmark of algorithmically-rejected trading events from a live Solana decentralized-exchange filter stack
  • The benchmark contains 6,659 rejection events linked to 169,122 post-rejection price and liquidity observations and 1,836 graveyard-tracker snapshots
  • Outcome labels follow the five-tier classification of Kamat (2026c): saved (windowed), saved (early-death), missed, flat, and unclassifiable
  • Thresholds use the trough-to-reference and peak-to-reference price ratios within a 24-hour window

The Value of Information: A Puzzle(https://arxiv.org/abs/2605.11180)

GeomHerd: A Forward-looking Herding Quantification via Ricci Flow Geometry on Agent Interactive Simulations(https://arxiv.org/abs/2605.11645)

  • The paper proposes GeomHerd, a forward-looking geometric framework to quantify herding in financial markets.
  • GeomHerd uses a heterogeneous LLM-driven multi-agent simulator to generate agent-interaction graphs.
  • The framework evaluates the geometric pipeline on the Cividino--Sornette continuous-spin agent-based substrate as the headline financial testbed.
  • GeomHerd captures the structural topology of emerging coordination by tracking the discrete Ollivier--Ricci curvature of action graphs.
  • The paper establishes a mean-field bridge mapping the graph-theoretic metric to CSAD, the classical macroscopic herding statistic.
  • Empirically, GeomHerd anticipates herding long before aggregate market baselines, with the primary detector firing a median of 272 steps before order-parameter onset.
  • A contagion detector recalls 65% of critical trajectories 318 steps early, and the agent-graph signal precedes price-correlation-graph baselines by 40 steps.
  • The geometric signature transfers out-of-domain to the Vicsek self-driven-particle model, and a curvature-conditioned forecasting head reduces cascade-window log-return MAE.

  • The analysis is computationally straightforward, extending conjugate-form Bayesian analyses for sequential filtering and model monitoring.
  • The paper highlights the improvements achievable in asset price forecasting relative to standard models using detailed empirical studies of multiple S&P sector ETFs.
  • The model delivers contextual insights on the nature and practical relevance of volatility leverage and feedback effects.

Overparametrized models with posterior drift(https://arxiv.org/abs/2506.23619)

  • The paper investigates the impact of posterior drift on out-of-sample forecasting accuracy in overparametrized machine learning models.
